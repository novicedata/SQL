  <summary><strong> 2️⃣ TCL </strong></summary>

- **TCL**: Transaction Control Language. 트랜잭션을 제어하는 명령어
  - **트랜잭션**: 쪼개질 수 없는 업무처리의 단위
  - **트랜잭션의 특징**
    - 1. 원자성: 트랜잭션으로 묶인 일련의 동작들은 모두 성공하거나 실패해야 함
      2. 일관성: 트랜잭션이 완료된 후에도 데이터 베이스가 가진 데이터에 일관성이 있어야 함
      3. 고립성: 하나의 트랜잭션은 고립되어 수행되어야 함
      4. 지속성: 트랜잭션이 성공적으로 수행되었을 경우 트랜잭션이 변경한 데이터가 영구적으로 저장되어야 함 

- **COMMIT**: 데이터 변경된 내용을 확정, 반영하는 명령어. 데이터 파일에 기록이 되고 트랜잭션이 완료됨
- **ROLLBACK**: 데이터 변경된 내용을 취소하는 명령어. 이전 값으로 복구 됨.
- **SAVEPOINT**: ROLLBACK을 수행할 때 전체 작업을 되돌리지 않고 일부만 되돌릴 수 있게 하는 기능을 가진 명령어

<details>
  <summary><strong> 3️⃣ DDL </strong></summary>

- **DDL**: Data Definition Language. 데이터 정의하는 SQL

- **CREATE**: 테이블 생성 명령어
  > <img width="668" height="285" alt="image" src="https://github.com/user-attachments/assets/1acddfe9-2f81-45e4-8893-3c53ae4b030c" />

- **CREATE시 규칙**
  - 1. 테이블명은 고유해야함
    2. 한 테이블 내에서 칼럼명은 고유해야함
    3. 칼럼명 뒤에 데이터 유형과 데이터 크기가 명시되어야 함
    4. 칼럼에 대한 정의는 괄호 안에 기술
    5. 각 칼럼들은 콤마로 구분
    6. 테이블 명과 칼럼 명은 숫자로 시작될 수 없음
    7. 마지막은 세미콜론으로 끝남
   
- **CREATE시 제약조건**
  - **PRIMAET KEY**: 테이블에 저장된 각각의 Row에 대한 고유성 보장. 한 테이블에 하나씩 정의. NULL값 허용 X
  - **UNIQUE KEY**: PRIMARY KEY와 유사하게 테이블에 저장된 각각의 Row에 대한 고유성 보장하기 위한 제약 조건. 하지만 NULL값이 허용
  - **NOT NULL**: 해당 칼럼에 NULL 값이 입력되는 것 허용 X
  - **CHECK**: 칼럼에 저장될 수 있는 값 범위 제한
    > <img width="620" height="68" alt="image" src="https://github.com/user-attachments/assets/9fce76cf-033a-43e2-9b04-39920ecc05c5" />
  - **FOREIGN KEY**: 하나의 테이블이 다른 테이블을 참조하고자 할 때 FK를 정의해 줌.
 
- **CREATE시 참조 무결성 관련 옵션**
  - **CASCADE**: 부모 값 삭제 시 자식 값 같이 삭제
  - **SET NULL**: 부모 값 삭제 시 자식의 해당 칼럼 NULL 처리
  - **SET DEFAULT**: 부모 값 삭제 시 자식의 해당 칼럼 DEFAULT 값으로 변경
  - **RESTRICT**: 자식 테이블 해당 데이터가 PK로 존재하지 않는 경우에만 부모 값 삭제 및 수정 가능
  - **NO ACTION**: 참조 무결성 제약이 걸려 있는 경우 삭제 및 수정 불가
 
- **ALTER**
  - **ADD COLUMN**: `ALTER TABLE 테이블명 ADD 칼럼명 데이터 유형;`
  - **DROP COLUMN**: `ALTER TABLE 테이블명 DROP COLUMN 칼럼명;`
  - **RENAME COLUMN**: `RENAME 기존 테이블명 TO 변경할 테이블명;`
  - **TRUNCATE TABLE**: `TRUNCATE TABLE 테이블 명` [ROLLBACK 안됨]

- **DCL**: Data Control Language. USER를 생성하고 USER에게 데이터를 컨트롤 할 수 있는 권한을 부여하거나 회수하는 명령어

  - **CREATE USER**: 사용자 생성

    > `ALTER USER 사용자명 IDENTIFIED BY 패스워드;`

  - **DROP USER**: 사용자 삭제
    > `DROP USER 사용자명;`
